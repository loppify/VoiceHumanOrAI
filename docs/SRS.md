# ДОДАТОК Б. Специфікація програмного забезпечення (SRS)

## 1. ВСТУП

### 1.1 Огляд продукту
Програмний продукт є системою бінарної класифікації мовного сигналу, що дозволяє визначити походження аудіозапису за критерієм "Людина-ШІ". Система використовує гібридний підхід, що поєднує інноваційний біонічний метод (базований на координатно-топологічному відображенні фонем) та класичні алгоритми машинного навчання (Random Forest/SVM).

### 1.2 Мета
Метою розробки є автоматизація процесу ідентифікації згенерованого мовлення, зниження ризиків від технологій Deepfake та надання інструменту для порівняльного аналізу ефективності біонічного та ML-методів розпізнавання.

### 1.3 Межі
Система працює локально (десктоп або локальний сервер). Обробляє аудіофайли виключно у форматі WAV. Не призначена для обробки аудіо у режимі реального часу (стрімінг). 

## 2. ЗАГАЛЬНИЙ ОПИС

### 2.1 Перспективи продукту
Система може бути використана експертами-криміналістами, дослідниками мовних технологій та як компонент безпеки для верифікації голосової автентифікації.

### 2.2 Функції продукту
- Завантаження та попередня обробка WAV-файлів (22050 Hz, Mono).
- Виділення просторових ознак та пошук центрів щільності у фонетичному просторі Хелвага-Щерби.
- Класифікація за вирішальним правилом біонічного методу (поріг $R = 16$).
- Екстракція MFCCs, Chroma, Spectral Rolloff та класифікація через ML-модель.
- Візуалізація часового графіка сигналу та трикутника щільності.
- Формування підсумкового звіту-вердикту на дашборді.

### 2.3 Характеристики користувачів
- **Дослідники / Експерти:** Потребують детальної візуалізації та розуміння механізмів прийняття рішень. Знайомі з основами обробки сигналів.
- **Звичайні користувачі:** Потребують лише кінцевого результату ("Людина" чи "ШІ").

### 2.4 Загальні обмеження
- Обсяг завантажуваного аудіофайлу не повинен перевищувати 20 МБ (~1-2 хвилини).
- Точність ML-моделі сильно залежить від репрезентативності тренувальної вибірки (навчального датасету).

## 3. КОНКРЕТНІ ВИМОГИ

### 3.1 Вимоги до зовнішніх інтерфейсів
#### 1. Інтерфейс користувача
- Веб-дашборд на базі Dash/Plotly з темною темою оформлення (DARKLY).
- Елемент Drag-and-Drop для завантаження аудіо.
- Два окремі блоки з результатами (Біоніка та ML).

#### 2. Апаратний інтерфейс
- Процесор: x86-64 або ARM64 (рекомендується від 4 ядер).
- Оперативна пам'ять: мінімум 4 ГБ.

#### 3. Програмний інтерфейс
- ОС: Windows / Linux / macOS.
- Середовище виконання: Python 3.9+.
- Основні бібліотеки: `scipy`, `numpy`, `librosa`, `scikit-learn`, `dash`, `plotly`.

### 3.2 Атрибути програмного продукту
#### 1. Надійність
- Система повинна коректно обробляти помилки читання битих аудіофайлів, виводячи відповідне повідомлення без збою роботи застосунку.

#### 2. Швидкодія (Продуктивність)
- Аналіз 10-секундного аудіофайлу повинен займати не більше 5 секунд на процесорі середнього класу.

#### 3. Супровіджуваність
- Код повинен бути структурованим (розділеним на логічні модулі: ядро біоніки, ядро ML, інтерфейс) та містити докладні docstrings, що дозволить легко оновлювати алгоритми.
