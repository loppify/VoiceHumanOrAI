## 2. ІНСТРУКЦІЯ (ПРОМПТ) ДЛЯ ШІ
*Нижче наведено максимально деталізований промпт. Скопіюй весь текст у рамці нижче та надішли його ChatGPT (GPT-4o), Claude 3.5 Sonnet або Gemini 1.5 Pro в новому чаті.*

```text
Ти — Senior Software Engineer, Data Scientist та експерт з архітектури ПЗ. Твоє завдання — розробити повноцінний, професійний код та архітектурну документацію для Комплексного Курсового Проєкту (який згодом стане дипломною роботою). 
Тема: "Програмна система бінарної класифікації мовного сигналу за критерієм «Людина-ШІ»".

КОНТЕКСТ ТА ВИМОГИ:
Система повинна аналізувати аудіофайли (WAV, 22050 Hz, 8/16-bit, Mono, ~10 сек) і визначати їхнє походження. Щоб проєкт був науково цінним та отримав найвищий бал, система повинна містити ДВА незалежних модулі класифікації для порівняльного аналізу:

МОДУЛЬ 1: Біонічний метод (Основний алгоритм розпізнавання образів)
- Базується на побудові відображення звуків у фонетичному просторі трикутника голосних Хелвага-Щерби.
- Сигнал ділиться на ділянки T=2500 семплів. Використовується згладжування (smoothing window = 16).
- Аналіз осциляцій (find_peaks) для виділення ознак v1, v2.
- Локалізація: пошук 3-х центрів максимальної щільності точок (алгоритм "ковзного вікна"). Точки локалізовані, якщо Евклідова відстань між ними мінімальна. 
- Між 3 центрами обчислюються відстані R12, R13, R23.
- Вирішальне правило: Якщо середнє R > 16, це "Людина" (через природну варіативність артикуляції кластери розмиті). Якщо R <= 16, це "ШІ" (математично ідеальна генерація фонем).

МОДУЛЬ 2: Класичний Machine Learning (ML алгоритм)
- Екстракція ознак за допомогою `librosa`: MFCCs (13-40 коефіцієнтів), Chroma STFT, Spectral Rolloff.
- Модель: Scikit-Learn Random Forest або SVM з RBF-ядром (бо датасет мікроскопічний і Deep Learning призведе до overfitting).
- Крос-валідація, розрахунок Accuracy та F1-Score.

ІНТЕРФЕЙС (Dash / Plotly):
- Темна тема (dbc.themes.DARKLY).
- Можливість завантажити WAV файл через UI.
- Відмальовка часового графіка сигналу.
- Відмальовка Scatter Plot (трикутник голосних з 3 центрами щільності) для біонічного методу.
- Виведення результатів (вердикт Біонічного методу + вердикт ML методу).

ЩО ТОБІ ПОТРІБНО ЗГЕНЕРУВАТИ ЗАРАЗ (У ВІДПОВІДІ):

1. Структуру проєкту (професійне дерево папок `src`, `models`, `ui`, `data`).
2. Повний код `bionic_core.py` (математика біонічного методу на numpy/scipy).
3. Повний код `ml_core.py` (витягування фіч librosa та клас-обгортка для sklearn моделі).
4. Повний код `app.py` (застосунок Dash).
5. UML-діаграми у форматі PlantUML:
   - Use Case Diagram.
   - Component Diagram (показує зв'язок UI з BionicCore та MLCore).
6. Структуру Специфікації Вимог (SRS) за стандартом IEEE 830-1993 (Вступ, Загальний опис, Конкретні вимоги), яку я зможу вставити в Додаток Б мого звіту.

Код має бути написаний ідеально: згідно з PEP8, з type hints та детальними docstrings. Зроби архітектуру модульною та легкою для масштабування.
```
