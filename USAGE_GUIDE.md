# Інструкція з використання гібридної системи VoiceHumanOrAI

Цей посібник допоможе вам налаштувати та використовувати систему для генерації датасетів, навчання моделей та аналізу аудіофайлів.

---

## 1. Генерація Датасету (dataset_builder.py)

Скрипт автоматично завантажує реальні людські голоси та генерує їхні ШІ-копії (клони) за допомогою вибраного провайдера.

### Доступні провайдери:
1.  **`edge`** (за замовчуванням) — Використовує хмарний Microsoft Edge TTS. Швидко, але без клонування.
2.  **`lux`** — Локальне клонування через LuxTTS. Потребує GPU (або CPU) та встановлених залежностей в `external/LuxTTS`.
3.  **`mosstts`** — Локальна генерація через MOSS-TTS-Nano. Оптимізовано для CPU.
4.  **`voicebox`** — Інтеграція з Voicebox Studio. Потребує запущеного сервера Voicebox на порту 17493.
5.  **`omnivoice`** — Інтеграція з OmniVoice Studio. Потребує запущеного сервера OmniVoice на порту 8000.

### Приклади команд:
```bash
# Проста генерація 10 пар через Edge TTS
python dataset_builder.py --samples 10 --provider edge

# Клонування через LuxTTS (використовує реальні голоси як референс)
python dataset_builder.py --samples 50 --provider lux

# Використання іншого датасету з HuggingFace
python dataset_builder.py --dataset "librispeech_asr" --config "clean" --samples 100
```

---

## 2. Навчання ML-моделі (train_model.py)

Після того, як папки `dataset/human` та `dataset/ai` наповнені файлами, потрібно навчити класифікатор.

```bash
python src/train_model.py
```
Скрипт проведе крос-валідацію та збереже навчену модель у папку `models/`.

---

## 3. Веб-інтерфейс (app.py)

Запуск інтерактивного дашборду для аналізу:

```bash
python src/app.py
```
Після запуску відкрийте браузер за адресою: `http://127.0.0.1:8050`.

---

## 4. Використання зовнішніх студій (Voicebox / OmniVoice)

Для роботи провайдерів `voicebox` або `omnivoice` у `dataset_builder.py`, ви повинні спочатку запустити їхні бекенди:

- **Для Voicebox:** Перейдіть в `external/voicebox` та запустіть `just dev` (якщо встановлено Just) або `python backend/main.py`.
- **Для OmniVoice:** Перейдіть в `external/OmniVoice-Studio` та запустіть `python backend/main.py`.

---

## 5. Поради для наукового дослідження

1. **Порівняння:** Згенеруйте 50 пар через `edge` та 50 пар через `lux`. Порівняйте точність біонічного методу на цих групах.
2. **Калібрування:** Використовуйте слайдер "Чутливість (Поріг R)" в UI, щоб знайти оптимальну межу для нових моделей клонування.
3. **XAI:** Звертайте увагу на графік "Важливість ознак". Якщо модель реагує лише на "Spectral Rolloff", можливо, вона вивчила різницю у якості запису, а не саму природу голосу.
